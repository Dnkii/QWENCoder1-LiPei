# 系统架构设计

## 整体架构

保险理赔助手系统采用模块化设计，主要包含以下几个核心模块：

1. 文档分类模块
2. 关键字提取模块
3. 责任评估模块
4. 主流程编排模块

## 模块详细设计

### 1. 文档分类模块 (DocumentClassifier)

- 功能：对上传的文档或影像进行智能分类
- 输入：文档路径
- 输出：文档类别
- 技术实现：结合大模型和传统分类算法

### 2. 关键字提取模块 (KeywordExtractor)

- 功能：从分类后的文档中提取预设的关键字
- 输入：文档路径、文档类别
- 输出：关键字及其值的映射
- 技术实现：基于规则和大模型的混合方法

### 3. 责任评估模块 (LiabilityEvaluator)

- 功能：根据保险合同对案件进行责任判定
- 输入：理赔数据（包含提取的关键字）、合同条款
- 输出：责任评估结果
- 技术实现：规则引擎结合大模型推理

### 4. 主流程编排模块 (InsuranceClaimAssistant)

- 功能：整合各模块功能，提供统一接口
- 输入：理赔文档路径、保险合同条款
- 输出：完整的理赔处理结果
- 技术实现：流程控制和数据传递

## 数据流

1. 用户上传理赔文档
2. 文档分类模块对文档进行分类
3. 关键字提取模块根据类别提取相应关键字
4. 责任评估模块基于提取的关键字和合同条款进行责任判定
5. 返回最终的理赔处理结果

## 技术栈

- 编程语言：Python
- 大模型API：OpenAI GPT系列或其他大模型
- 文档处理：PyPDF2, OpenCV等
- Web框架：FastAPI（可选，用于提供API服务）
- 数据库：SQLite/PostgreSQL（可选，用于存储处理结果）
